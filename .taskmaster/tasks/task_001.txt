# Task ID: 1
# Title: Design Database Schema for Queue System
# Status: pending
# Dependencies: None
# Priority: high
# Description: Create comprehensive PostgreSQL database schema for analysis request queue system with optimized JSONB indexing and proper status tracking.
# Details:
Design PostgreSQL database schema with three core tables: 1) analysis_request_queue (UUID primary key, request_data JSONB with jsonb_path_ops GIN index, status enum, priority int, created_at/updated_at timestamps, retry_count int, processing_started_at timestamp), 2) processing_status (UUID primary key, request_id FK, step_name varchar, status enum, step_data JSONB, completed_at timestamp), 3) results_archive (UUID primary key, original_request_id FK, final_report JSONB, email_sent boolean, archived_at timestamp). Implement proper JSONB indexing using jsonb_path_ops for containment queries and expression indexes for frequently queried paths like (request_data->>'company_name') and (request_data->>'ticker'). Add composite indexes on (status, priority, created_at) for optimal polling performance. Use TOAST storage optimization for large JSONB documents.

# Test Strategy:
Validate schema creation with PostgreSQL 15+ compatibility, test JSONB query performance with sample data (target <50ms for polling queries), verify foreign key constraints, test concurrent insert/update scenarios, validate index usage with EXPLAIN ANALYZE.
